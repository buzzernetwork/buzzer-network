==================================================

**Note**: This is the initial PRD (v1). For the refined and current requirements, see `r04_prd_v2.txt`. This document is kept for historical reference.

HIGH-LEVEL USER JOURNEY & REQUIREMENTS

1. OVERVIEW
-----------
Buzzer Network is an X402-based decentralized ad network on BASE blockchain that connects advertisers directly with premium publishers, enabling crypto-native payments and transparent revenue sharing.

Core Value Proposition:
- Advertisers: Lower fees, instant payments, direct publisher relationships
- Publishers: Higher revenue share, crypto payments, quality-focused network
- Network: BUZZ token economy, governance, sustainable growth

2. USER JOURNEYS - HIGH LEVEL
-------------------------------

A. PUBLISHER JOURNEY
---------------------

Step 1: Discovery & Onboarding
- Publisher discovers Buzzer Network (website, referral, marketing)
- Clicks "Become a Publisher" 
- Creates account (wallet connect or email)
- Fills basic info (website URL, category, traffic volume)

Step 2: Verification & Approval
- System verifies domain ownership (DNS record or HTML file)
- Quality check: Traffic audit, content review, brand safety
- Approval/rejection decision
- If approved: Onboarding complete, publisher dashboard access

Step 3: Integration
- Publisher receives integration code snippet (X402 compliant)
- Adds code to website (header, body, or specific ad slots)
- Tests ad display
- Goes live

Step 4: Earnings & Payments
- Ads served on publisher site
- Impressions tracked and verified (oracle-verified)
- Earnings accrue in real-time dashboard
- Automatic crypto payments (daily/weekly) to publisher wallet
- Publisher can withdraw anytime (minimum threshold)

Step 5: Optimization
- Publisher views analytics (impressions, revenue, CTR)
- Adjusts ad placements for better performance
- Receives recommendations
- Manages multiple sites from one dashboard

B. ADVERTISER JOURNEY
---------------------

Step 1: Account Creation
- Advertiser discovers platform
- Creates account (wallet connect)
- Completes profile (company info, payment preferences)

Step 2: Campaign Creation
- Click "Create Campaign"
- Define campaign goal (awareness, clicks, conversions)
- Set targeting (demographics, geography, interests)
- Upload ad creative(s) or use template
- Set budget (total budget, daily budget, bid amount)

Step 3: Campaign Launch
- Review campaign settings
- Fund campaign wallet (ETH, USDC, or BUZZ tokens)
- Activate campaign
- Campaign goes live immediately

Step 4: Campaign Management
- Monitor real-time performance (impressions, clicks, spend)
- Adjust targeting or bids in real-time
- Pause/resume campaigns
- Update ad creatives
- View detailed analytics

Step 5: Payment & Settlement
- Smart contract escrows campaign funds
- Payments automatically deducted per impression/click
- Remaining funds can be withdrawn anytime
- Receive invoices/receipts on-chain

C. NETWORK OPERATIONS (INTERNAL)
--------------------------------

Step 1: Quality Assurance
- Automated publisher quality scoring
- Fraud detection for clicks/impressions
- Content moderation for ad creatives
- Brand safety monitoring

Step 2: Matching & Delivery
- Real-time ad selection algorithm
- Match advertiser campaigns to publisher inventory
- Deliver ads via X402 protocol
- Track and verify all events

Step 3: Settlement
- Aggregate impression/click data
- Calculate publisher earnings
- Calculate network fees
- Execute automated payments
- Update on-chain records
